# psy-slots

Telegram-бот записи к психологу с напоминаниями. Serverless-стек: **aiogram** (webhook) + **YDB** на **Yandex Cloud Functions**.

## Как это работает

- **Клиент** пишет боту → `/start` показывает welcome и свободные слоты **на сегодня** (или сообщение о невозможности брони, если слотов нет) → подтверждает запись. Может посмотреть свои записи и отменить их. Навигация — постоянная reply-клавиатура (главное меню).
- **Слоты эфемерны**: нигде не хранятся, а вычисляются на лету из интервалов доступности и длительности слота. Бронировать можно только слоты на сегодня; на кнопке слот показан диапазоном «14:00-14:20».
- **Бронь** — самостоятельная персистентная строка в YDB с неизменным UUID и **явными `start`/`end` в UTC** (денормализованы при создании, чтобы позднее изменение длительности не сдвигало существующие записи). Отмена — soft-переход состояния, никогда не hard-delete.
- **Администратор** (`ADMIN_TELEGRAM_ID`) видит все активные брони (клиент показан человекочитаемо: `Имя Фамилия (@username)` → `@username` → `telegram_id`, имя берётся через `bot.get_chat`) и может отменить любую — клиент при этом получает уведомление. Когда бронь отменяет сам клиент, уведомление об этом получает администратор.
- **Напоминания** шлются за фиксированные 5 минут до начала брони отдельной функцией по timer-триггеру. Идемпотентны: маркер по `booking_id` гарантирует одно напоминание даже при повторных тиках.
- **Время везде в UTC**, показывается клиенту в фиксированной таймзоне **`Europe/Moscow`** (не настраивается).

## Архитектура

Serverless накладывает ограничения — функция эфемерна, никакого long polling и фоновых задач:

- **`app/handler.py`** — точка входа webhook-функции. Валидирует секрет Telegram, десериализует апдейт, прогоняет через aiogram `Dispatcher`. Один апдейт на вызов, состояние между вызовами не удерживается.
- **`app/notify.py`** — точка входа timer-функции. На каждый тик находит созревшие брони и шлёт напоминания через `ReminderService`.
- Клиент YDB (`app/ydb_client.py`) и `Bot`/`Dispatcher` — тёплые синглтоны на уровне модуля, переиспользуются тёплым инстансом.
- Все переходы состояния брони — через **единый доменный сервисный слой**, не инлайн в хендлерах. Хендлеры тонкие; доменная логика тестируется без Telegram.

### Структура

```
app/
  handler.py            # webhook-функция (точка входа YC)
  notify.py             # timer-функция напоминаний (точка входа YC)
  config.py             # типизированная загрузка env (pydantic-settings)
  ydb_client.py         # тёплый драйвер + пул сессий YDB
  migrations.py         # идемпотентный раннер DDL (CREATE TABLE IF NOT EXISTS)
  domain/               # чистое ядро: без YDB и Telegram
    booking.py          #   модель брони и доменные ошибки
    slots.py            #   генерация эфемерных слотов (перевод локаль→UTC, политика DST)
  services/             # доменный сервисный слой над YDB
    booking_service.py  #   создание/отмена/чтение броней (сериализуемые транзакции)
    slot_service.py     #   оркестрация свободных слотов (вычитает занятые/прошедшие)
    reminder_service.py #   отправка напоминаний с идемпотентностью
  bot/                  # тонкий слой aiogram
    __init__.py         #   сборка Bot/Dispatcher, внедрение сервисов
    handlers/           #   клиентский и админ-роутеры
    keyboards.py        #   инлайн-клавиатуры, главное меню + (де)сериализация callback_data
    naming.py           #   человекочитаемое имя клиента через bot.get_chat
    formatting.py       #   форматирование UTC→Europe/Moscow для показа
tests/                  # тесты (pytest), покрытие ≥ 80%
openspec/               # спеки изменений (spec-driven workflow)
```

### Таблицы YDB (`app/migrations.py`)

- `bookings` — брони (`id` UUID PK, `client_id`, `start_utc`/`end_utc`, `status`, вторичный индекс по `start_utc`).
- `sent_reminders` — маркеры отправленных напоминаний (`booking_id` PK) для идемпотентности.

Профиля клиента нет: таймзона фиксирована (`Europe/Moscow`), а имена берутся из Telegram на лету через `bot.get_chat` — таблица `clients` не создаётся.

## Команды бота

Дублируются кнопками постоянного меню: «✏️ Записаться» → `/book`, «🗓️ Мои записи» → `/mybookings`, «📘 Показать все брони» → `/admin` (последняя видна только админу).

| Команда        | Действие                                                        |
|----------------|-----------------------------------------------------------------|
| `/start`       | Welcome (`WELCOME_MESSAGE`) + слоты на сегодня, либо `BOOKING_UNAVAILABLE_MESSAGE`; показывает меню |
| `/book`        | Показать свободные слоты на сегодня                              |
| `/mybookings`  | Активные записи клиента (с кнопкой отмены)                       |
| `/admin`       | Все активные брони (только для `ADMIN_TELEGRAM_ID`)             |

## Конфигурация

Всё — только из переменных окружения (см. `.env.example`). Отсутствие обязательной переменной или недопустимое значение — ошибка на этапе инициализации.

| Переменная                     | Обяз. | Описание                                                        |
|--------------------------------|:-----:|-----------------------------------------------------------------|
| `BOT_TOKEN`                    |   ✓   | Токен Telegram-бота                                             |
| `WEBHOOK_SECRET`               |   ✓   | Секрет для проверки заголовка `X-Telegram-Bot-Api-Secret-Token` |
| `ADMIN_TELEGRAM_ID`            |   ✓   | Telegram ID администратора                                      |
| `YDB_ENDPOINT`                 |   ✓   | gRPC-эндпоинт YDB                                               |
| `YDB_DATABASE`                 |   ✓   | Путь к базе YDB                                                 |
| `AVAILABILITY_INTERVALS`       |   ✓   | Интервалы доступности (в `Europe/Moscow`), напр. `10:00-14:00,15:00-18:00` (только время суток, без дня недели, без пересечений) |
| `WELCOME_MESSAGE`              |   ✓   | Текст приветствия                                              |
| `BOOKING_UNAVAILABLE_MESSAGE`  |   ✓   | Текст, когда свободных слотов на сегодня нет                    |
| `SLOT_DURATION_MINUTES`        |       | Длительность слота, минут (по умолчанию `20`)                   |

Креды YDB драйвер читает из окружения (`ydb.credentials_from_env_variables()`) — например, `YDB_SERVICE_ACCOUNT_KEY_FILE_CREDENTIALS` или метаданные сервисного аккаунта в YC.

## Разработка

Стек: **Python 3.14**, пакетный менеджер/раннер — **`uv`**. Весь код и инструменты запускаются через `uv run`.

```bash
uv sync                 # установить зависимости
cp .env.example .env    # заполнить переменные

make test               # pytest + покрытие (порог 80%)
make validate           # ruff format + ruff check + mypy (strict)
```

Обе команды должны быть зелёными перед завершением любой задачи.

Применить миграции схемы (идемпотентно, вне горячего пути вебхука):

```bash
uv run python -m app.migrations
```

### Требования к коду

- **Test-first**: сначала падающий тест, затем реализация. Тесты — в `tests/`.
- Покрытие ≥ 80%.
- Публичные функции типизированы; mypy `strict` без `# type: ignore`-заглушек.
- Секреты и токены не хардкодятся и не коммитятся.

### Рабочий процесс

Spec-driven (OpenSpec): сначала спека изменения (`openspec/`), потом код. Не отступать от активной спеки без её обновления.

## Деплой (Yandex Cloud Functions)

Две функции с общим кодом и переменными окружения:

1. **Webhook** — точка входа `app.handler.handler`, HTTP-триггер. Зарегистрировать URL в Telegram через `setWebhook` с `secret_token = WEBHOOK_SECRET`.
2. **Напоминания** — точка входа `app.notify.notify`, timer-триггер (напр. раз в минуту).

Зависимости для рантайма — в `requirements.txt` (сгенерирован из `pyproject.toml` через `uv pip compile`). Миграции применяются отдельным шагом, не из хендлера.
