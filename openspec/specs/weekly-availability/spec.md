## Requirements
### Requirement: Повторяющееся недельное расписание в таймзоне специалиста

Система SHALL хранить доступность специалиста как набор интервалов, каждый из которых задаёт день недели, время начала и время конца в **локальном настенном времени таймзоны специалиста**. Интервалы MUST трактоваться как повторяющиеся еженедельно. Время интервалов MUST NOT храниться в UTC — привязка к конкретным датам и перевод в UTC выполняется позже при генерации слотов.

#### Scenario: Интервал сохраняется в локальном времени

- **WHEN** сервис добавляет специалисту интервал (напр. вторник 10:00–14:00)
- **THEN** сохраняются день недели и время суток начала/конца, и последующее чтение возвращает те же локальные значения

#### Scenario: Расписание повторяется еженедельно

- **WHEN** у специалиста задан интервал на некоторый день недели
- **THEN** он применяется к каждому вхождению этого дня недели, без привязки к конкретной календарной дате

### Requirement: Добавление интервала с валидацией

Доменный сервис SHALL добавлять интервал только после валидации: день недели корректен, время начала строго меньше времени конца, обе границы лежат в пределах суток, и интервал ссылается на существующего специалиста. Каждый добавленный интервал MUST получать неизменный идентификатор.

#### Scenario: Валидный интервал добавляется

- **WHEN** сервис добавляет корректный интервал существующему специалисту
- **THEN** создаётся строка с сгенерированным идентификатором, и интервал появляется в списке специалиста

#### Scenario: Начало не раньше конца

- **WHEN** сервис получает интервал, где время начала больше или равно времени конца
- **THEN** операция завершается доменной ошибкой валидации, интервал не сохраняется

#### Scenario: Некорректный день недели или границы суток

- **WHEN** сервис получает интервал с днём недели вне допустимого диапазона или с временем вне пределов суток
- **THEN** операция завершается доменной ошибкой валидации, интервал не сохраняется

#### Scenario: Интервал для несуществующего специалиста

- **WHEN** сервис пытается добавить интервал специалисту, которого нет в реестре
- **THEN** операция завершается доменной ошибкой «специалист не найден», интервал не сохраняется

### Requirement: Запрет пересечения интервалов в один день недели

Система SHALL запрещать сохранение интервала, пересекающегося по времени с уже существующим интервалом того же специалиста в тот же день недели. Смежные интервалы (конец одного равен началу другого) MUST быть разрешены.

#### Scenario: Пересекающийся интервал отклоняется

- **WHEN** сервис добавляет интервал, диапазон которого пересекается с существующим интервалом того же специалиста в тот же день недели
- **THEN** операция завершается доменной ошибкой пересечения, новый интервал не сохраняется

#### Scenario: Смежные интервалы допускаются

- **WHEN** сервис добавляет интервал, начало которого равно концу существующего интервала того же дня недели (без перекрытия)
- **THEN** интервал сохраняется успешно

### Requirement: Удаление и список интервалов

Доменный сервис SHALL позволять удалять интервал по его идентификатору и возвращать список всех интервалов специалиста. Расписание MUST быть редактируемым; список MUST корректно работать при отсутствии интервалов.

#### Scenario: Удаление интервала

- **WHEN** сервис удаляет существующий интервал по идентификатору
- **THEN** интервал исчезает из списка специалиста

#### Scenario: Пустое расписание

- **WHEN** у специалиста нет ни одного интервала
- **THEN** список возвращает пустой результат без ошибки

#### Scenario: Список по специалисту

- **WHEN** у специалиста есть несколько интервалов в разные дни недели
- **THEN** список возвращает ровно его интервалы и не возвращает интервалы других специалистов
