# bot-config Specification

## Purpose
Бот обслуживает **одного** психолога. Его параметры и тексты берутся из переменных окружения функции — не из БД и не хардкодом: длительность слота, интервалы доступности (только время суток, без дня недели), тексты сообщений, идентификатор администратора. Здесь же — их парсинг и валидация. Таймзона — исключение: она фиксирована как `Europe/Moscow` и не настраивается.

## Requirements

### Requirement: Единственный психолог, параметры из окружения

Система SHALL обслуживать ровно одного психолога и брать все его параметры из переменных окружения. Система MUST NOT хранить сущность/таблицу специалистов и MUST NOT содержать enum, константы или иную привязку к числу специалистов. Значения параметров MUST NOT быть захардкожены в коде.

#### Scenario: Параметры читаются из окружения

- **WHEN** боту нужны длительность слота, интервалы доступности или тексты
- **THEN** они берутся из переменных окружения, без обращения к таблице специалистов и без захардкоженных констант

#### Scenario: Нет реестра специалистов

- **WHEN** система работает
- **THEN** в ней отсутствует сущность/таблица специалистов и любые ссылки на специалиста по идентификатору

### Requirement: Длительность слота из окружения с дефолтом

Длительность слота SHALL читаться из переменной окружения. При отсутствии переменной значение MUST быть равно **20 минутам** по умолчанию. Заданное значение MUST быть строго положительным; ноль, отрицательное или нечисловое значение MUST завершаться явной ошибкой конфигурации.

#### Scenario: Значение задано

- **WHEN** переменная длительности задана положительным числом минут
- **THEN** используется это значение

#### Scenario: Значение не задано

- **WHEN** переменная длительности не задана
- **THEN** используется дефолт 20 минут

#### Scenario: Недопустимое значение отклоняется

- **WHEN** переменная длительности равна нулю, отрицательна или не число
- **THEN** загрузка конфигурации завершается явной ошибкой, дефолт не подставляется

### Requirement: Интервалы доступности из окружения без дня недели

Интервалы доступности SHALL читаться из переменной окружения как список интервалов времени суток (например `10:00-14:00,15:00-18:00`). День недели MUST NOT быть частью интервала — одни и те же интервалы применяются к каждому дню. В каждом интервале время начала MUST быть строго меньше времени конца, обе границы MUST лежать в пределах суток. Интервалы одного списка MUST NOT пересекаться. Нарушение MUST завершаться явной ошибкой конфигурации.

#### Scenario: Список интервалов парсится

- **WHEN** переменная задаёт корректный список интервалов времени суток
- **THEN** он разбирается в набор интервалов, применяемых одинаково к каждому дню

#### Scenario: День недели не учитывается

- **WHEN** заданы интервалы доступности
- **THEN** они трактуются как время суток без привязки к дню недели

#### Scenario: Недопустимый интервал отклоняется

- **WHEN** интервал имеет начало не раньше конца, границы вне суток, либо пересекается с другим интервалом списка
- **THEN** загрузка конфигурации завершается явной ошибкой валидации

### Requirement: Фиксированная таймзона Europe/Moscow

Система SHALL использовать фиксированную таймзону `Europe/Moscow` для трактовки интервалов доступности и для показа времени пользователю. Таймзона MUST NOT читаться из окружения и MUST NOT настраиваться пользователем или психологом. Время в БД по-прежнему хранится в UTC.

#### Scenario: Интервалы трактуются в Europe/Moscow

- **WHEN** система материализует интервалы доступности в конкретные моменты
- **THEN** локальное время интервалов трактуется в таймзоне `Europe/Moscow`

#### Scenario: Показ времени в Europe/Moscow

- **WHEN** боту нужно показать пользователю время слота, брони или напоминания
- **THEN** время конвертируется из UTC в `Europe/Moscow`

#### Scenario: Переменная TIMEZONE не требуется

- **WHEN** загружается конфигурация без переменной `TIMEZONE`
- **THEN** загрузка завершается успешно, используется фиксированная `Europe/Moscow`

### Requirement: Тексты сообщений из окружения

Текст приветственного сообщения и текст сообщения о невозможности брони («на сегодня свободных слотов нет») SHALL читаться из переменных окружения. Эти тексты MUST NOT быть захардкожены в коде.

#### Scenario: Тексты берутся из окружения

- **WHEN** боту нужно показать приветствие или сообщение о невозможности брони
- **THEN** используется соответствующий текст из переменной окружения

### Requirement: Идентификатор администратора из окружения

Система SHALL читать идентификатор Telegram-администратора (психолога) из переменной окружения. Он используется для авторизации админ-потока. Значение MUST NOT быть захардкожено.

#### Scenario: Идентификатор администратора задан

- **WHEN** система проверяет доступ к админ-действиям
- **THEN** доступ сверяется с идентификатором из переменной окружения
