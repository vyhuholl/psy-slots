## Why

Проекту нужен исполняемый каркас до любой доменной логики: точка входа Telegram-бота, работающая в webhook-режиме на Yandex Cloud Functions, и тёплый клиент YDB. Без этого фундамента ни один доменный change (специалисты, расписание, брони) нельзя ни задеплоить, ни протестировать end-to-end. Объединяем транспорт (webhook) и персистентность (YDB) в один bootstrap, т.к. поодиночке они бесполезны: бот без хранилища ничего не помнит, хранилище без входа недостижимо.

## What Changes

- Точка входа YC Cloud Function: принимает HTTP-событие вебхука, отдаёт корректный ответ, передаёт апдейт в aiogram 3.x (**webhook-режим**, никакого polling).
- Идемпотентный, stateless-хендлер: не держит фонового состояния и переживает повторную доставку одного апдейта без двойного эффекта.
- Валидация секретного токена вебхука Telegram; отклонение запросов без/с неверным токеном.
- Тёплый клиент YDB на уровне модуля: инициализируется один раз, переиспользуется между вызовами, вручную не закрывается.
- Механизм миграций схемы YDB (идемпотентное создание таблиц), запускаемый явно, а не на горячем пути хендлера.
- Загрузка всех секретов (токен бота, webhook-секрет, креды и endpoint YDB) только из переменных окружения функции.
- Скелетный хендлер `/start` — подтверждает, что маршрут «Telegram → функция → aiogram → ответ» работает целиком.

## Capabilities

### New Capabilities
- `bot-webhook`: точка входа YC-функции в webhook-режиме — разбор события, валидация webhook-секрета, идемпотентная передача апдейта в aiogram, ответ Telegram, конфигурация из env, скелет `/start`.
- `persistence`: тёплый модульный клиент YDB, конфигурация подключения из env, идемпотентный механизм миграций схемы.

## Impact

- **Новый код**: пакет `app/` — точка входа функции, сборка aiogram Dispatcher/Bot, модуль клиента YDB, модуль миграций, чтение конфигурации из env.
- **Тесты**: `tests/` — разбор события вебхука, отклонение неверного секрета, идемпотентность, загрузка конфигурации, миграции против локального/поддельного YDB.
- **Зависимости**: `aiogram>=3.29`, `ydb>=3.30` (уже в `pyproject.toml`).
- **Инфраструктура (вне кода)**: YC Cloud Function, реестр контейнеров/деплой, serverless-инстанс YDB, переменные окружения функции. Регистрация вебхука в Telegram — операционный шаг.
- **Последующие changes**: `add-specialist-registry`, `add-weekly-availability`, `add-booking-lifecycle` и остальные опираются на клиент YDB и механизм миграций отсюда; доменные хендлеры подключаются к Dispatcher, созданному здесь.
