## ADDED Requirements

### Requirement: Точка входа функции обрабатывает событие вебхука

Функция Yandex Cloud SHALL предоставлять единственный обработчик HTTP-события, который принимает POST-запрос вебхука Telegram, десериализует тело в объект `Update` aiogram, передаёт его в Dispatcher и возвращает HTTP-ответ, завершая вызов. Обработчик MUST работать без long polling.

#### Scenario: Валидный апдейт обрабатывается

- **WHEN** функция получает HTTP-событие с корректным JSON-телом апдейта Telegram и верным секретом вебхука
- **THEN** тело разбирается в `Update`, передаётся в Dispatcher, и функция возвращает HTTP 200

#### Scenario: Некорректное тело не роняет функцию

- **WHEN** функция получает событие с телом, которое не является валидным JSON апдейта Telegram
- **THEN** функция не выбрасывает необработанное исключение и возвращает HTTP 200 без обработки апдейта

### Requirement: Валидация секретного токена вебхука

Обработчик SHALL проверять заголовок `X-Telegram-Bot-Api-Secret-Token` против секрета из окружения и отклонять запросы с отсутствующим или неверным секретом, не передавая их в Dispatcher.

#### Scenario: Верный секрет

- **WHEN** входящий запрос содержит заголовок секрета, равный сконфигурированному значению
- **THEN** запрос принимается и апдейт передаётся в Dispatcher

#### Scenario: Неверный или отсутствующий секрет

- **WHEN** входящий запрос не содержит заголовок секрета или содержит значение, не совпадающее с сконфигурированным
- **THEN** апдейт не передаётся в Dispatcher и функция возвращает ответ с кодом отклонения (HTTP 403)

### Requirement: Идемпотентная stateless-обработка

Обработчик SHALL быть без фонового состояния между вызовами и не запускать планировщики или фоновые asyncio-задачи, переживающие завершение вызова. Повторная доставка одного и того же `update_id` MUST NOT вызывать двойной наблюдаемый эффект.

#### Scenario: Повторная доставка апдейта

- **WHEN** Telegram доставляет апдейт с уже обработанным ранее `update_id`
- **THEN** повторная обработка не производит второй наблюдаемый побочный эффект

#### Scenario: Нет фоновых задач

- **WHEN** обработчик завершает вызов
- **THEN** не остаётся запланированных фоновых задач или планировщиков внутри функции

### Requirement: Конфигурация из переменных окружения

Токен бота и секрет вебхука SHALL читаться исключительно из переменных окружения функции. Значения MUST NOT быть захардкожены в коде. При отсутствии обязательной переменной инициализация SHALL завершаться явной ошибкой.

#### Scenario: Секреты отсутствуют

- **WHEN** обязательная переменная окружения (токен бота или секрет вебхука) не задана
- **THEN** инициализация завершается явной ошибкой конфигурации, а не работает с пустым значением

### Requirement: Скелетный хендлер /start

Dispatcher SHALL регистрировать хендлер команды `/start`, отправляющий пользователю приветственное сообщение, подтверждая сквозную работу маршрута «Telegram → функция → aiogram → ответ».

#### Scenario: Пользователь отправляет /start

- **WHEN** пользователь отправляет команду `/start`
- **THEN** бот отвечает приветственным сообщением
