## ADDED Requirements

### Requirement: Профиль клиента с таймзоной

Система SHALL хранить минимальный профиль клиента в YDB, ключом которого является идентификатор пользователя Telegram, и который содержит таймзону клиента как корректный IANA-идентификатор. Профиль SHALL читаться и обновляться через доменный сервис.

#### Scenario: Сохранение таймзоны клиента

- **WHEN** сервис устанавливает клиенту корректную IANA-таймзону
- **THEN** значение сохраняется и возвращается при следующем чтении профиля

#### Scenario: Невалидная таймзона отклоняется

- **WHEN** сервис получает строку таймзоны, не являющуюся корректным IANA-идентификатором
- **THEN** операция завершается доменной ошибкой валидации, профиль не изменяется

### Requirement: Запрос таймзоны при неизвестном профиле

Когда клиент взаимодействует с ботом впервые и его таймзона неизвестна, система SHALL предложить установить таймзону до показа времени. После установки таймзона MUST переиспользоваться без повторного запроса.

#### Scenario: Первый контакт запрашивает таймзону

- **WHEN** клиент без сохранённой таймзоны начинает поток записи
- **THEN** бот предлагает выбрать/установить таймзону, и после выбора она сохраняется в профиль

#### Scenario: Известная таймзона не запрашивается повторно

- **WHEN** клиент с уже сохранённой таймзоной начинает поток записи
- **THEN** таймзона берётся из профиля без повторного запроса
