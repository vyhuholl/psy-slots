## Why

Доменный слой (конфигурация, слоты, брони) готов, но у клиента нет способа записаться. Это первый пользовательский поток в Telegram: увидеть свободные слоты на сегодня, подтвердить бронь, а также посмотреть и отменить свои записи. Психолог один, запись возможна только на сегодня, поэтому выбор специалиста и дня не нужен. Всё время показывается в часовом поясе клиента (требование CLAUDE.md), поэтому вводится минимальный профиль клиента с таймзоной.

## What Changes

- Поток записи на тонких aiogram-хендлерах: приветствие → свободные слоты на сегодня → подтверждение; доменные вызовы — через существующие сервисы (`slot`, `booking`), без логики инлайн.
- **Приветственное сообщение** при старте — текст из конфигурации (env).
- **Навигация без серверного состояния**: выбор слота закодирован в `callback_data` инлайн-кнопок (никакого FSM/сессий) — корректно для эфемерной serverless-функции.
- **Профиль клиента с таймзоной**: при первом обращении таймзона запрашивается и сохраняется; всё время показывается сконвертированным в неё.
- **Нет свободных слотов на сегодня** → показать сообщение о невозможности брони (текст из конфигурации), без ошибки.
- Подтверждение создаёт бронь через доменный сервис; **занятый к этому моменту слот** (гонка) обрабатывается дружелюбно, без сбоя и без второй брони.
- Просмотр своих активных броней и их **отмена** (soft, через сервис) — тоже во времени клиента.
- Устойчивость к повтору: повторное подтверждение не создаёт вторую бронь (защита на стороне `booking-lifecycle`).

## Capabilities

### New Capabilities
- `client-profile`: минимальная запись клиента в YDB (telegram id, таймзона) и её чтение/установка; запрос таймзоны при первом обращении.
- `client-booking-flow`: клиентский Telegram-поток — приветствие из конфигурации, показ свободных слотов на сегодня без серверного состояния, сообщение о невозможности брони, подтверждение брони, просмотр и отмена своих броней, отображение времени в таймзоне клиента.

## Impact

- **Новый код**: `app/bot/handlers/client.py` (тонкие хендлеры), `app/bot/keyboards.py` (инлайн-клавиатуры + схема `callback_data`), `app/bot/formatting.py` (форматирование времени в TZ клиента), `app/domain/client.py` + `app/services/client_service.py` (профиль/таймзона), регистрация таблицы `clients` в `app/migrations.py`, подключение роутеров к Dispatcher.
- **Тесты**: `tests/` — хендлеры с замоканными сервисами (правильные вызовы, клавиатуры, форматирование в TZ, приветствие, «нет слотов»), профиль клиента (запрос/сохранение таймзоны), обработка занятого слота и отмены.
- **Зависит от**: `bot-config` (тексты, TZ), `add-slot-generation`, `add-booking-lifecycle`, `bootstrap-bot` (Dispatcher/webhook).
- **Разблокирует / переиспользуется**: `add-reminder-notifications` использует таймзону клиента для текста напоминаний.
- **Замечание по объёму**: `client-profile` добавлен здесь как минимально необходимый для показа времени; при желании выносится в отдельный change.
