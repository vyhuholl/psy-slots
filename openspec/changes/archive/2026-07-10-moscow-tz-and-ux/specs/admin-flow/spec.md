## MODIFIED Requirements

### Requirement: Просмотр активных броней

Администратор SHALL видеть список активных броней со временем, сконвертированным в `Europe/Moscow`. Каждая бронь SHALL показывать клиента человекочитаемым именем (см. требование «Человекочитаемое имя клиента»), а не техническим `client <id>`.

#### Scenario: Список броней

- **WHEN** администратор запрашивает брони
- **THEN** бот показывает активные брони со временем в `Europe/Moscow` и человекочитаемым именем клиента на каждой кнопке

#### Scenario: Пустой список

- **WHEN** активных броней нет
- **THEN** бот сообщает об отсутствии записей без ошибки

## ADDED Requirements

### Requirement: Человекочитаемое имя клиента

Система SHALL формировать человекочитаемое имя клиента для показа администратору, получая данные пользователя через `bot.get_chat`. Формат MUST быть следующим по приоритету:
1. Есть имя и username → `Имя Фамилия (@username)`;
2. Нет имени, есть username → `@username`;
3. Нет username → `telegram_id`.
Если данные получить не удалось (например, клиент недоступен), система MUST деградировать до `telegram_id` без ошибки. В кнопке брони имя ставится после диапазона без обрамляющих скобок, например «14:20-14:40 Иван Иванов (@ivanov) ❌».

#### Scenario: Имя и username

- **WHEN** у клиента есть имя (и, возможно, фамилия) и username
- **THEN** имя формируется как `Имя Фамилия (@username)`

#### Scenario: Только username

- **WHEN** у клиента нет имени, но есть username
- **THEN** имя формируется как `@username`

#### Scenario: Только telegram_id

- **WHEN** у клиента нет username
- **THEN** имя формируется как `telegram_id`

#### Scenario: Данные недоступны

- **WHEN** получить данные клиента через `bot.get_chat` не удалось
- **THEN** имя деградирует до `telegram_id`, без ошибки в потоке администратора
