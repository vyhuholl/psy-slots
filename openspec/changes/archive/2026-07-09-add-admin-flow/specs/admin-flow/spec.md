## ADDED Requirements

### Requirement: Авторизация администратора по конфигурации

Команды админ-потока SHALL быть доступны только пользователю Telegram, чей идентификатор совпадает с `ADMIN_TELEGRAM_ID` из конфигурации. Любой другой пользователь MUST получать отказ и MUST NOT видеть управляющих действий.

#### Scenario: Администратор получает доступ

- **WHEN** пользователь с `telegram_id`, равным `ADMIN_TELEGRAM_ID`, открывает админ-меню
- **THEN** ему показываются управляющие действия

#### Scenario: Прочий пользователь получает отказ

- **WHEN** пользователь, чей `telegram_id` не равен `ADMIN_TELEGRAM_ID`, вызывает админ-команду
- **THEN** бот отвечает отказом, управляющие действия не показываются

### Requirement: Просмотр активных броней

Администратор SHALL видеть список активных броней со временем, сконвертированным в таймзону конфигурации.

#### Scenario: Список броней

- **WHEN** администратор запрашивает брони
- **THEN** бот показывает активные брони со временем в таймзоне конфигурации

#### Scenario: Пустой список

- **WHEN** активных броней нет
- **THEN** бот сообщает об отсутствии записей без ошибки

### Requirement: Отмена брони администратором

Администратор SHALL отменять бронь через доменный сервис как soft-переход (без hard-delete). При отмене клиент MUST быть уведомлён. Повторная отмена уже отменённой брони MUST быть идемпотентной.

#### Scenario: Отмена и уведомление клиента

- **WHEN** администратор отменяет активную бронь
- **THEN** бронь переводится в состояние «отменена» через сервис, и клиенту отправляется уведомление об отмене

#### Scenario: Отмена уже отменённой брони

- **WHEN** администратор отменяет бронь, уже находящуюся в состоянии «отменена»
- **THEN** операция завершается без ошибки и без повторного эффекта (идемпотентно)
