## ADDED Requirements

### Requirement: Тёплый модульный клиент YDB

Система SHALL инициализировать клиент YDB (driver + пул сессий) один раз на уровне модуля и переиспользовать его между вызовами тёплого инстанса функции. Клиент MUST NOT создаваться заново на каждый вызов и MUST NOT закрываться вручную в хендлере.

#### Scenario: Переиспользование между вызовами

- **WHEN** тёплый инстанс функции обрабатывает несколько вызовов подряд
- **THEN** используется один и тот же экземпляр драйвера YDB без повторной инициализации

#### Scenario: Ленивая готовность драйвера

- **WHEN** модуль импортируется, а затем выполняется первый запрос
- **THEN** драйвер дожидается готовности перед выполнением запроса и переиспользуется далее

### Requirement: Конфигурация подключения из переменных окружения

Endpoint, database path и креды YDB SHALL читаться исключительно из переменных окружения функции. Значения MUST NOT быть захардкожены. При отсутствии обязательной переменной инициализация SHALL завершаться явной ошибкой.

#### Scenario: Параметры подключения заданы

- **WHEN** заданы переменные окружения endpoint, database и учётных данных YDB
- **THEN** клиент конфигурируется этими значениями без обращения к захардкоженным константам

#### Scenario: Обязательная переменная отсутствует

- **WHEN** обязательная переменная подключения YDB не задана
- **THEN** инициализация клиента завершается явной ошибкой конфигурации

### Requirement: Идемпотентный механизм миграций схемы

Система SHALL предоставлять явно запускаемую операцию миграции, создающую требуемые таблицы YDB, если их ещё нет. Операция MUST быть идемпотентной (повторный запуск на уже мигрированной БД безопасен) и MUST NOT выполняться на горячем пути хендлера вебхука.

#### Scenario: Первый запуск создаёт схему

- **WHEN** миграция запускается против БД без требуемых таблиц
- **THEN** таблицы создаются, и операция завершается успешно

#### Scenario: Повторный запуск безопасен

- **WHEN** миграция запускается против БД, где таблицы уже существуют
- **THEN** операция завершается успешно без ошибки и без разрушения существующих данных

#### Scenario: Миграция вне горячего пути

- **WHEN** функция обрабатывает вебхук-апдейт
- **THEN** миграция схемы в рамках обработки апдейта не запускается
