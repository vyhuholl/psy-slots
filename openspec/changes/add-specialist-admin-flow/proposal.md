## Why

Психолог должен сам управлять своими параметрами: длительностью слота, недельным расписанием, а также видеть и отменять записи. Сервисы для этого уже есть, но нет Telegram-интерфейса специалиста и нет узла, где реализуется инвариант «изменение расписания или длительности не должно затирать уже забронированные слоты — их судьбу решать явно». Этот change собирает управляющий поток и композит-гвард поверх существующих сервисов. Для авторизации специалиста нужна привязка его записи к Telegram-аккаунту.

## What Changes

- Привязка специалиста к аккаунту Telegram: у записи специалиста появляется `telegram_id` и поиск по нему — расширение `specialist-registry`.
- Админ-поток на тонких aiogram-хендлерах (навигация без серверного состояния, как в клиентском потоке): изменение длительности, редактирование расписания (добавление/удаление интервалов), просмотр броней, отмена брони.
- **Авторизация**: команды доступны только привязанному специалисту; чужой/непривязанный пользователь — отказ.
- **Гвард инвариантности** (композит поверх read-запросов броней): изменение длительности и удаление интервала расписания **блокируются, пока есть активные будущие брони**, которых это касается — специалист должен сначала их отменить (явное разрешение «требовать отмены»). Добавление интервала безопасно.
- Отмена брони специалистом — soft-переход через сервис, с уведомлением клиента.
- Время специалисту показывается в его таймзоне.

## Capabilities

### New Capabilities
- `specialist-admin-flow`: Telegram-поток специалиста — авторизация, изменение длительности и расписания с гвардом по активным броням, просмотр и отмена броней, показ времени в таймзоне специалиста.

### Modified Capabilities
- `specialist-registry`: добавляется привязка записи специалиста к аккаунту Telegram (`telegram_id`) и поиск специалиста по `telegram_id` для авторизации.

## Impact

- **Новый код**: `app/bot/handlers/admin.py` (тонкие хендлеры), расширение `app/bot/keyboards.py` (клавиатуры специалиста + `callback_data`), `app/services/schedule_guard.py` или методы в существующих сервисах для гварда, расширение `specialist_service`/`app/domain/specialist.py` (`telegram_id`, `get_by_telegram_id`), ALTER `specialists` в `app/migrations.py`, подключение админ-роутеров к Dispatcher.
- **Тесты**: авторизация (привязанный/непривязанный), изменение длительности (успех и блокировка при активных бронях), add/remove интервала (успех, блокировка remove при бронях), просмотр/отмена, уведомление клиента при отмене.
- **Зависит от**: `add-specialist-registry` (сущность+сервис), `add-weekly-availability` (расписание), `add-booking-lifecycle` (read-запросы + отмена), `add-client-booking-flow` (общие `app/bot/` утилиты: форматирование, клавиатуры, `callback_data`), `bootstrap-bot` (Dispatcher).
- **Замыкает инвариант**: гвард «длительность/расписание vs брони», отложенный в `specialist-registry`, `weekly-availability` и `booking-lifecycle`, реализуется здесь.
