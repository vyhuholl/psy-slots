## ADDED Requirements

### Requirement: Привязка специалиста к аккаунту Telegram

Запись специалиста SHALL опционально содержать идентификатор аккаунта Telegram (`telegram_id`), однозначно связывающий психолога с его записью. Один и тот же `telegram_id` MUST NOT быть привязан более чем к одному специалисту. Доменный сервис SHALL предоставлять поиск специалиста по `telegram_id`.

#### Scenario: Поиск специалиста по telegram_id

- **WHEN** сервис запрашивает специалиста по привязанному `telegram_id`
- **THEN** возвращается соответствующий специалист

#### Scenario: Непривязанный telegram_id

- **WHEN** сервис запрашивает специалиста по `telegram_id`, не привязанному ни к одной записи
- **THEN** возвращается явный результат «не найдено», без утечки исключения хранилища

#### Scenario: Повторная привязка одного telegram_id отклоняется

- **WHEN** сервис пытается привязать `telegram_id`, уже закреплённый за другим специалистом
- **THEN** операция завершается доменной ошибкой, привязка не изменяется
